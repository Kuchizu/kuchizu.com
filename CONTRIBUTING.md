# Contributing

1. Fork the repo
2. Create a branch (`git checkout -b feature/amazing`)
3. Make changes
4. Test locally with `docker-compose -f docker/docker-compose.yml up`
5. Commit (`git commit -m "feat: add amazing feature"`)
6. Push and open a PR

## Commit Convention

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code refactoring
- `ci:` CI/CD changes
